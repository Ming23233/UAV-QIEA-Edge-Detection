# UAV-QIEA Lightweight UAV Small-Object Detection

This repository contains the pre-submission research code for an edge-constrained UAV small-object detection study based on a YOLOX/ByteTrack-style detector. The project evaluates a high-resolution P2 detection branch and a quantum-inspired evolutionary algorithm (QIEA) for lightweight structure candidate search.

The central claim of the study is not that QIEA always gives the highest final detection accuracy. The evidence supports a conservative conclusion: the P2 branch provides stable small-object detection gains, while QIEA is useful for searching and screening lightweight architectural candidates under accuracy and deployment-efficiency constraints. Full-training verification is needed because proxy-search rankings may differ from final 100-epoch rankings.

This is a code release for paper submission. It intentionally does not include datasets, model checkpoints, raw logs, pretrained weights, or large detection-output files. The `results/` directory contains only small manuscript-aligned CSV/JSON/Markdown summary artifacts for reviewer inspection.

## Repository Layout

```text
.
|-- configs/                 Search-space and experiment config files
|-- docs/                    Dataset and reproducibility notes
|-- examples/                Example command scripts
|-- results/                 Lightweight manuscript-aligned result summaries
|-- scripts/                 Experiment, search, evaluation, and artifact scripts
|-- src/                     Research scaffold modules
`-- third_party/ByteTrack/   Modified YOLOX/ByteTrack code used by experiments
```

Runtime outputs are written under `outputs/`, which is ignored by Git.

## Main Components

- **P2 high-resolution branch**: adds a stride-4 detection path for dense UAV small-object scenes.
- **QIEA search**: encodes optional lightweight modules and training hyperparameters as probability vectors, then updates candidate distributions using proxy fitness.
- **Search comparison**: compares QIEA with random search, GA, and an SA/QUBO heuristic under the same proxy budget.
- **Deployment evidence**: reports parameters, FLOPs, GPU latency, and memory usage for edge-oriented analysis.
- **External case study scripts**: include AU-AIR and other UAV-domain engineering-case workflows.

## Installation

Create a Python environment and install the repository dependencies:

```bash
conda create -n uav-qiea python=3.11 -y
conda activate uav-qiea
pip install -r requirements.txt
pip install -r third_party/ByteTrack/requirements.txt
pip install -e third_party/ByteTrack
```

Install a PyTorch build matching your CUDA driver from the official PyTorch instructions.

## Dataset Preparation

This repository does not include datasets. Prepare a COCO-style VisDrone dataset and pass paths explicitly:

```text
data/
`-- visdrone_det_coco/
    |-- annotations/
    |   |-- instances_train.json
    |   |-- instances_val.json
    |   |-- search_train.json
    |   `-- search_val.json
    `-- images/
```

VisDrone and UAVDT conversion helpers are under `third_party/ByteTrack/tools/`. The proxy search uses `search_train.json` and `search_val.json`; these can be generated from the converted COCO annotations with the repository scripts.

## Key Reproduction Commands

Run the stable baseline and P2 multi-seed training:

```bash
python scripts/run_visdrone_100ep_correct_flow.py \
  --python python \
  --dataset-root /path/to/visdrone_det_coco \
  --epochs 100 \
  --batch-size 8 \
  --amp
```

Generate the discrete search-space metadata:

```bash
python scripts/stage3_generate_search_space.py
```

Run QIEA proxy search:

```bash
python scripts/run_stage5_proxy_search.py \
  --python python \
  --dataset-root /path/to/visdrone_det_coco
```

Run QIEA-Final full training:

```bash
python scripts/run_stage6_qiea_final.py \
  --python python \
  --dataset-root /path/to/visdrone_det_coco \
  --epochs 100 \
  --batch-size 8 \
  --amp
```

Run search-algorithm comparison:

```bash
python scripts/run_stage19_search_algorithm_comparison.py \
  --python python \
  --dataset-root /path/to/visdrone_det_coco
```

Run full 100-epoch validation for best search candidates:

```bash
python scripts/run_stage26_best_candidates_seed42.py \
  --python python \
  --dataset-root /path/to/visdrone_det_coco \
  --epochs 100 \
  --batch-size 8 \
  --data-num-workers 0 \
  --amp
```

Run efficiency and deployment-oriented evaluation:

```bash
python scripts/run_stage20_edge_efficiency_eval.py
```

Some efficiency scripts expect checkpoints generated by previous training stages. Update checkpoint paths or pass script arguments according to your local layout.

Recompute scale statistics and diagnostic error counts when local annotation and detection files are available:

```bash
python scripts/dataset_statistics.py \
  --dataset VisDrone=/path/to/visdrone_det_coco \
  --out-dir outputs/dataset_statistics

python scripts/diagnose_small_object_errors.py \
  --gt /path/to/visdrone_det_coco/annotations/instances_val.json \
  --detections baseline.json plus_p2.json qiea_final.json \
  --model-names Baseline +P2 QIEA-Final \
  --subset dense-small \
  --out-csv outputs/dense_small_error_diagnosis.csv
```

## Results

The lightweight result artifacts in `results/` mirror the current manuscript tables: multi-seed VisDrone results, scale statistics, diagnostic small-object AP, dense-small error diagnosis, proxy search comparison, Stage26 full-training verification, deployment-cost records, AU-AIR case results, and marginal accuracy-cost analysis.

The repository does not bundle raw datasets, checkpoints, full logs, or large detection JSON files. Those are intentionally excluded because of dataset licensing, file size, and local-path hygiene.
